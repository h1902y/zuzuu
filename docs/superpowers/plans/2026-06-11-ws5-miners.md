# WS5 — Multi-Faculty Miners — Implementation Plan

> Part of the Faculty+Evolution Program (final workstream). TDD; zero-dep; Node ≥22; deterministic (golden tests, no LLM in the hermetic suite); commit per task; trailer `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`; explicit `git add`. Baseline: 276 tests.

**Goal:** Close the evolve loop for all faculties. Today only Knowledge has trace-miners (`mns/knowledge/distill.mjs`). Add a miner registry + per-faculty miners (Actions, Guardrails, Instructions; Memory = stub) so a real session trace yields ranked proposals across faculties: capture → mine → eval-rank (WS4) → propose (spine, WS2) → human batch-approve → mint generation (WS3).

## Task WS5-T1: miner registry + superset mineTranscript + driver
**Create** `mns/miners/{registry,knowledge}.mjs`; **Modify** `mns/knowledge/distill.mjs` (extend `mineTranscript` to a superset), `mns/commands/distill.mjs` (`--all-faculties` driver). **Test** `tests/unit/miners-registry.test.mjs`.
- Extend `mineTranscript(file)` to ALSO surface (superset — existing keys unchanged so the Knowledge miner + `distill.test.mjs` stay green): `sequences` (consecutive Bash command n-grams within a session), `correctionTurns` (user turns w/ corrective lexicon following an assistant tool action), `destructiveFailures` (failed tool calls whose input matches a destructive shape). Read the transcript's `uses[]`/results already parsed.
- `mns/miners/registry.mjs` — `register({faculty, aggregate, propose})`, `all()`. Each miner: `aggregate(sessions, opts) → [{candidate|payload, evidence}]`, `propose(mnsDir, aggregated) → count` (files into that faculty's inbox/proposals via the spine).
- `mns/miners/knowledge.mjs` — thin: re-export the EXISTING `aggregate` from distill.mjs + a `propose` that calls the existing knowledge proposal path. Register it. (No behavior change to knowledge mining.)
- `mns/commands/distill.mjs` — add `--all-faculties`: mine each transcript once (superset), fan to each registered miner's `aggregate`, score via the WS4 eval lens, `propose` per faculty; report per-faculty counts. Default (no flag) = knowledge only (back-compat).
- TDD: registry round-trip; superset `mineTranscript` still emits the original `commands/files/failures` keys (distill.test green) + the new keys on a fixture; `--all-faculties` driver fans to registered miners.

## Task WS5-T2: Actions miner
**Create** `mns/miners/actions.mjs`; **Test** `tests/unit/miner-actions.test.mjs`.
- `aggregate`: detect a recurring **command sequence** — the same ordered Bash n-gram (n≥2) appearing ≥3× across ≥2 sessions (from `sequences`). Also read `.mns/live/actions.jsonl` (the trail) for ad-hoc repeats.
- `propose`: scaffold a **runbook** action into `actions/inbox/<slug>/` (reuse `act-author.mjs`'s `proposeAction`): `SKILL.md` = the sequence as numbered steps; `action.json` = inferred title/promptSnippet. Carry `evidence` + an `eval.json` (score) alongside.
- TDD (golden, deterministic): a fixture with a repeated 2-command sequence across 2 sessions → one action proposal in inbox; a one-off sequence → none.

## Task WS5-T3: Guardrails miner (highest risk — defensive)
**Create** `mns/miners/guardrails.mjs`; **Test** `tests/unit/miner-guardrails.test.mjs`.
- `aggregate`: from `destructiveFailures` — a destructive-shaped command (`rm -rf`, `git push --force`, `DROP TABLE`, `chmod -R`) that **failed ≥3× across ≥2 sessions**, OR a near-miss against the guardrails decision log. **Patterns are literal-escaped from the observed command** (never free regex). **Default `action:'ask'`, never `deny`.**
- `propose`: a guardrail proposal `{kind:'rule', payload:{id,action:'ask',tool,pattern,reason}}` into `guardrails/proposals/` (spine) → flows through `mns review` → on approve appends to rules.json (WS2 guardrails adapter).
- **Cross-session corroboration REQUIRED** (a single angry session must not mint a rule). Eval lens down-weights single-session signals.
- TDD: a destructive cmd failing across 2 sessions → one `ask` rule proposal with a literal-escaped pattern; a single-session destructive event → NO proposal (the key safety test).

## Task WS5-T4: Instructions miner + Memory stub + close
**Create** `mns/miners/instructions.mjs`, `mns/miners/memory.mjs`; **Test** `tests/unit/miner-instructions.test.mjs`.
- Instructions `aggregate`: recurring **correction pattern** (from `correctionTurns`) — corrective user lexicon ("no, don't", "actually use", "always", "stop") recurring across ≥2 sessions → an instructions amendment proposal `{kind:'block', payload:{text}}`. Default LOW confidence (eval). Lexicon is data (zero-LLM).
- Memory `memory.mjs`: a registered STUB — `{faculty:'memory', aggregate: () => [], propose: () => 0, stub:true}`. Document (comment) what it WOULD mine (completed-session episodes → curated entries). Build nothing.
- TDD: a recurring correction across 2 sessions → one instructions proposal (low confidence); memory miner registers + is a no-op.
- Final: `mns distill --all-faculties` on a multi-faculty fixture surfaces proposals for ≥2 faculties; the LOG entry (exp-13) for the whole program (WS1–WS5) — append to `experiments/LOG.md`.

## Verification
- `npm test` green each task (276 + new; deterministic, zero-LLM). End-to-end: a real/fixture multi-session set through `mns distill --all-faculties` → ranked proposals across faculties; the single-session destructive case produces NO guardrail rule (safety); `mns review` shows them ranked; approving mints a generation (WS3).

## Risks
- **Guardrail false-positives** (highest): default `ask`, literal-escaped patterns, cross-session-gated, eval down-weights singletons, human gates every rule. The single-session-no-proposal test is mandatory.
- Miner thresholds are provisional — keep them in `aggregate(opts)` (tunable/testable), document as defaults.
- Instructions lexicon misfires → default low confidence + cross-session required.
