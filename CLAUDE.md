This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repository is

**A working early-stage build + its canonical design.** The host coding-agent (Claude Code / Codex / Gemini CLI / OpenCode / pi) supplies the **brain**; this project gives it evolving **modules** — Knowledge (semantic), Memory (episodic), Actions (procedural), Instructions (directive), Guardrails (protective, enforced) — that **graduate** across versioned generations, grown from the observability **trace** of real use, human-gated. We **wrap, serve, observe, evolve** a host we never drive. (Terminology: these five were called **faculties** until the `faculty → module` rename — code, CLI, README, and DESIGN now say **module**; `faculty` survives only as *intentional history* — the `commands/migrations/` shims, DESIGN.md's terminology notes, and LOG.md's append-only journal entries — not as drift to fix.)

**The v2 rebuild is now the product (rungs 1–8 shipped on `rebuild/kernel`, 2026-06-21).** The filesystem-native **note / module / project** model the redesign specs described is built and the v1 substrate is **deleted** — ~13k → ~3.6k lines. One coherent stack, filed by concept: `src/{notes,use,grow,guardrails,hosts,sessions,cli,serve}`. **Everything is an envelope** (markdown + frontmatter, distinguished by `type`): a *note* (one fact, optionally runnable) and a *module* manifest are the same shape. The four capability **verbs** — `query · act · check · review` — are CLI verbs over envelopes, dispatched through one registry (`enhance` was cut 2026-06-22 — its co-invocation miner was structurally dead; **observe** is the live proposal producer). The loop is **observe → propose → review → evolve** (review = the human gate's decision; evolve = write the note + mint a generation + log), every write **human-gated** (the gate is the moat). Built + verified: the substrate (the envelope parser, the `node:sqlite` graph index, the capability registry, content-addressed per-module generations), the loop (propose/review), **observe** (host-agnostic transcript mining → routed proposals, Design B — re-parse, never drive), the **CLI veneer** (`bin/zuzuu.mjs` → `cli/` → `api`), session management (session = git branch; worktrees + labels — the safety-critical `sessions/` engine, characterization-tested), the live **hook** (lifecycle + the enforced `PreToolUse` gate), and host **enablement**. The published binary runs the whole loop end to end. Check `docs/LOG.md` for the build journal and `docs/learn/` for the educative walk (lessons 02–08 cover the rebuild).

**Dropped, not ported:** the v1 OTLP/trace observability layer — v2's observe mines transcripts directly into proposals, so traces were never needed. **Still design-only / not built:** the *async* evolution service (Cloudflare Workflows); the cloud session waves (E container-per-worktree, F Fly local↔cloud sync, G portable/L7, H sharing) are infra-gated. Don't claim unbuilt parts work; don't treat shipped parts as absent — `docs/LOG.md` is the source of truth.

Naming: the project is **zuzuu** (current name; a return to the original concept) — was **motorsandsensors / mns** in the v0 phase (no `mns` alias remains — all backward-compat was removed). The **command is `zz`** (and `zuzuu`); the npm package is **`@zuzuucodes/cli`** (unscoped `zuzuu` is blocked by npm as too-similar to `zuul`; the `@zuzuucodes` org holds it). ## Architecture (the big picture)

Product code lives in **`src/`** (+ `web/`), filed **by concept, not by layer** (re-filed 2026-06-21). It's a plain DAG (no import cycles) with one invariant worth more than a layer diagram, now made **structural** (2026-06-28): **one writer — `grow/commit.mjs` is the SOLE note-writer + the loop's sole minter**, and the agent reaches it only through `review` (the gate); `use/` reads + runs — a run appends telemetry to the *log* (Data), never the notes. `commit` validates (the schema gate), logs, mints, and refuses `actor:'agent'`; `writeNote`/`removeNote` are called from `commit` and nowhere else (the operator-initiated *manifest* writes — `init`, the schema alter-table ops, on-demand template mint — bypass it deliberately, each a gate-event, and touch `module.md`/`project.md`, never a note). The atom is a **note** (`note › module › Project`). `src/README.md` is the map; lessons `docs/learn/02–08` walk it file by file. The nine dirs:

**`metal/` — the I/O owners (the deepest ring).** `git.mjs` is THE one git primitive (`git(args, cwd, input) → {ok, code, out, err}`, argv-array only / never a shell string / never throws — the rich `code` lets the session merge-tree probe tell a clean merge from a conflict from a fatal; **promoted byte-identical** from the session engine, which `sessions/git.mjs` now re-exports, so the safety-critical characterization stays exact). `fs.mjs` is the byte-I/O owner (`readText · writeText · remove · list · mkdirp`) — the one place note bytes cross the filesystem. `sqlite.mjs` is the lazy `node:sqlite` loader. Centralizing these gives the upper rings (the repo, the transaction boundary) one seam to wrap.

**`notes/` — the substrate (`note › module › project`).** `note.mjs` is the tolerant generic parser/serializer, kept PURE — no I/O (markdown body + frontmatter; only `type` required, unknown keys preserved round-trip-exact; `id` = the filename stem, injected by the caller, never in frontmatter — "everything is an envelope"). `repo.mjs` is the **note repository** — the one persistence seam over the pure transform: `readNote · writeNote · removeNote · listNoteIds`, where `writeNote` **validates before the write** (the schema gate — §validate). `rows.mjs` is the **lossless read projection** (`readEnvelopes`/`searchRows` hydrate the FULL on-disk envelope, so a custom frontmatter column survives to the grid — the index row carries only the indexed five). `validate.mjs` is the pre-write check on two axes: the per-type invariants AND — when a module declares a `fields` block — its **typed-column schema** (the 8 FieldTypes; a schema-violating note is rejected at the boundary; absent `fields` ⇒ schemaless). `store.mjs` resolves the git-citizen home (`git --show-toplevel` + `/.zuzuu` — never `git init`s) and any `module:id` address. `index.mjs` is a lazy `node:sqlite` cache (notes + KV props + a typed link graph + FTS5) whose `plan()` does **server-side** filter/sort/paginate over module-as-table (text/type/tag/status/`--where` EAV, parameterized SQL), rebuilt on mtime-staleness — the keystone bet, benchmarked at 5000 notes build 157ms / search 13ms. `module.mjs` reads `module.md` manifests; `project.mjs` reads the `project.md` manifest; `generation.mjs` is the content-addressed per-module **generations** (mint/`expandRollback`/read — a Data artifact with its operations, the same shape as `index.mjs`); `log.mjs` is the append-only mutation+run **journal**; `toon.mjs` = token-dense AXI output.

**`use/` — use the Project (read · run · inspect).** `query.mjs` (FTS + graph walk, TOON), `act.mjs` (run a runnable note under the guardrails gate + the `run.allow` command-axis — gated + allowlisted, not OS-sandboxed; capture `{stdout,stderr,exit,success}` + log), `check.mjs` (integrity: broken links · orphans · stale). These never mutate the Project.

**`grow/` — grow the Project: the loop (observe → propose → review → evolve), every write human-gated, through ONE boundary.** `commit.mjs` is **THE write boundary** (2026-06-28) — the sole note-writer + the loop's sole minter: `commit(home, {actor}, batch, opts)` applies a normalized op batch as ONE all-or-nothing transaction (per-op snapshot→restore compensation, never a blanket `git clean`), then mints **one generation per touched module**, and **refuses `actor:'agent'`** up front (the in-process trust gate — the agent's channel is observe→stage→review). `observe.mjs` (aggregate per-session signals across the corroboration threshold and **route** each candidate to the right module via the declarative `ROUTE` table — a recurring command → an `action`, a hot file → a `knowledge` entity, a repeated destructive command → an **ask** `guardrail`, a recurring correction → an `instruction` — reaching 4 of the 5 kinds; the live proposal producer), `propose.mjs` (the typed, deduped, ranked proposal queue), `review.mjs` (**THE human gate** — the *decision*; the only door to the Project), `evolve.mjs`/`plan.mjs` (the *execution* beat — they now feed `commit`, not their own writes). The graph-safe refactors `refactor.mjs` (`rename · merge · refactorField`) and `notes/generation`'s `expandRollback` became pure batch **expanders** — they DECIDE what changes (emit ops); `commit` decides HOW it lands. `schema.mjs` is the typed-column ALTER TABLE (`add/alter/dropColumn` — a `module.md` manifest write that mints its own generation, deliberately OUTSIDE `commit` since a manifest write IS the gate). *(The durable artifacts — `generation`, `log` — live in `notes/` as Data; rollback = restore-as-forward-motion, never `git revert`.)*

**`guardrails/` — `gate.mjs`**, the enforced `PreToolUse` check (rules are `type: rule` notes, severity deny>ask>allow, **fail-open**). It canonicalizes both **shell** and **file-write** tool names across hosts (`WRITE_ALIASES` → one `tool: write` rule fires everywhere), and a rule may set `match: path` to test only the file-path fields (not content). This is what makes the **brain write-protection** an enforced invariant: the seeded `protect-brain-writes`/`-shell` rules **deny the agent's direct writes to `.zuzuu/`** (worktrees excepted), and `protect-brain-exec` denies the agent shelling a `zz` WRITE verb via Bash (closing the Bash-bypass hole the in-process `actor` check can't — a shelled `zz` is a fresh operator process). That last rule's pattern is **generated from the command table** (`gateWriteVerbPattern` / `writeVerbPaths` in `cli/commands.mjs` — a verb is denied when it durably mutates AND isn't `agentInvokable`, so the sanctioned `stage`/`observe` stay allowed; aliases + a row's `writeSubcommands` like `review approve|apply` and `session land|drop` are followed), so the deny set **can't drift from the verb set**. The brain changes only through review — the agent reads its modules freely but can't author them (`zz stage` → approve is the only door).

**`hosts/` — observe a host (Design B).** `adapters/*` re-parse each host's real on-disk format → deterministic mining signals; `registry.mjs`/`capture.mjs`/`signals.mjs` are the host-blind core (iterate `detected()`, never a host name — adding a host = one adapter file + one registry line). `hook.mjs` maps every host's lifecycle → open/turn/end (OPEN grounds + opens the session branch; TURN checkpoints; END **finalizes and holds** the branch for the explicit merge gate — squash-merges only on the per-project `autoMerge` opt-in — then **observes**) and routes `PreToolUse` → the gate; fail-open, always exits 0.

**`sessions/` — session = git branch (a Layer-3 *surface* concern — the session surface's git plumbing, **not** a core lifecycle stage; consumed only by `cli/` + `hosts/hook`; the most safety-critical area, characterization-first).** `session-git.mjs` (one session = one `zz/session-*` branch, turn-checkpoints with secret exclusion, **END holds the branch (`finalizeSession` → `zz/held-*` + checkout base) behind the explicit merge gate** — `closeSession` squash-merges only via `zz session merge` or the `autoMerge` opt-in; merge/continue/discard resolve **both** namespaces by optional id; single-working-branch invariant; fail-soft, never throws — the lifecycle lives here, *not* in a separate index: the branch IS the record), `session-worktree.mjs` (a worktree per session → concurrency; `finalizeSessionWorktree` marks the held branch so a **live** agent is never mistaken for one awaiting merge), `git.mjs` (plumbing), `labels.mjs` (human labels).

**`cli/` — the thin veneer.** `commands.mjs` is the **declarative command table** — ONE row per command (`{ path, plane, summary, usage, handler, …metadata }`), the single source of truth FOUR consumers read: the router, `zz help` (folded from the table — an unlisted command can't exist), the gate deny set (`writeVerbPaths`), and the daemon catalog (`commandCatalog` → `zz commands --json`). `index.mjs` is the ~63-line router — parse argv → **longest-prefix** match against the table → build `ctx` → call the handler (owns no logic; it also stamps the trust boundary: the CLI process is the operator, the host hook the agent). The grammar is **two-tier** (Tier-1 flat hot-loop verbs · Tier-2 noun namespaces `note·gen·session·host·registry`); every old flat verb is a deprecating **alias row** (data, not a wrapper). `init.mjs` (git-citizen scaffold — the Project manifest + the `instructions` module seeded with the enforced safety floor (rule-notes, incl. the `.zuzuu/` write-protection) + best-practice guidance, and a `create` mutation logged per seed for provenance; the other content modules grow on demand), `session.mjs`, `enable.mjs` (install the hook block by a stable `#zz-hook` signature, never clobbering user hooks), `doctor.mjs` (health · status · explain), `code.mjs` (launch OpenCode as the bundled host), `web.mjs` (the workbench). AXI: TOON output, brief-by-default, no blocking prompts.

**`serve/` — compose · expose · ground.** `api.mjs` (the one façade — `open(cwd)` → a Project handle the CLI + web both consume; it stamps the `actor` and defaults it to `operator`), `dispatch.mjs` (the capability dispatch table — `register`/`invoke`, manifest-gated, fail-soft; `invoke` carries the trust `actor` and refuses a `write`-permission capability under `agent` — forward-looking, since no verb is `write` today), `wire.mjs` (`registerAll` — binds **`query · check · act · view · flow · validate`** through the one registry, so none skip the permission/actor gate; `review` deliberately unregistered — the human gate is not an agent-invokable capability), `digest.mjs` (the deterministic session-start brief).

**The method:** product code lives in `src/` (+ `web/`), full stop. The rebuild was **greenfield-kernel, cull-per-rung** — build v2 beside v1, keep the suite green at every commit, delete each v1 surface only after its v2 replacement proved out (rungs 1–8, `docs/LOG.md`). **Live experimentation happens OUTSIDE the repo** — `~/Documents/cards-game` is the dogfood target where zuzuu is exercised against real Claude Code sessions (init/enable/observe/the gate); findings that matter graduate into `docs/LOG.md` (the build journal, append-only). In-repo: `tests/playground/` = smoke vs real machine data (playground-5 mines real transcripts); `tests/unit/` = hermetic.

## Hard-won conventions (violating these has bitten us)

- **Real-wire-data rule:** adapters/integrations are built and verified against output the host *actually produced* — never from docs alone, never against self-invented fixtures (that's circular). Observe real events **before** wiring lifecycle semantics (docs lied twice: Claude `Stop` and OpenCode `session.idle` are per-*turn*, not end; OpenCode `session.deleted` is delete-only).
- **Golden ids in regression tests are pasted from a real run** — never hand-computed. If the id scheme changes intentionally, regenerate and review.
- **Verify fixes with `node bin/zuzuu.mjs`, never the PATH `zuzuu`** — the global is the *published* version and silently lacks unpushed code (a live bug-fix verification ran the old binary once and the bug "survived" its own fix).
- **Playground exit contract:** 0 = pass, **2 = skip** (host data absent — not a failure), anything else = fail. Don't "fix" skips to passes.
- **Hooks/plugins must never break the host:** always exit 0 (`… || true` wrappers, try-wrapped plugin), spawn detached, degrade silently. The guardrails **gate fails open** — engine/rule errors emit no decision (host's normal flow), never a block.
- **Home access is read-open but write-gated**: the agent **reads** `.zuzuu/` freely (a blanket `.zuzuu/**` deny would starve it of its own modules), but the **gate denies direct *writes*** to `.zuzuu/` (session worktrees excepted), so the brain changes only through review (`zz stage` → approve), never raw file writes. This lives as seeded gate rules (`protect-brain-writes`/`-shell`), not host settings-level denies — those stayed empty (`.zuzuu/.live/**` was dropped with the trace layer). The upgrade path for a pre-protection Project is `scripts/reconcile-brain.mjs` (re-init reconciles + flags `ungated` notes).
- **Secrets:** keys never land in tracked files; scan before commit/push. Generated host-enablement config (`.opencode/`, `.claude/settings*.json`) is git-ignored.

## Load-bearing vocabulary (these terms carry decisions)

- **Modules — the 5+3 anatomy** (the five, since 2026-06-10; renamed faculty→module later): a module is **generic** (any goal-shaped collection of notes — an **open set**, custom modules included); the **standard us-owned kinds** are Knowledge (semantic), Memory (episodic), Actions (procedural), **Instructions** (directive: the pinned steering/system-prompt artifact), Guardrails (protective: *enforced* tool gates) — examples with sensible defaults, each trace-grown + generation-pinned + served, **not a closed taxonomy**. **On-demand modules (2026-06-23, refined):** the five are just the standard *kinds*. `zz init` seeds **one module — `instructions`** — with the enforced safety floor (rule-notes, including the `.zuzuu/` write-protection) **plus best-practice instruction-notes** (how to use zuzuu well); the other three content modules (knowledge/memory/actions) start empty and **materialize on demand** (their `module.md` minted on first proposal, `src/notes/module-templates.mjs` + `grow/propose.mjs`) as the loop grows the Project. (The old `guardrails` module was folded into `instructions` at the rename — the gate enforces both for back-compat.) The types survive as vocabulary; only *shipping the content modules prebuilt* went away. **Cognition / Model / Workspace are host *anatomy*, not modules** (process / engine / arena; observed and steered, never graduated).
- **be / run / evolve**: what the agent *is* / what *serves & bounds* it / what *grows* it.
- **Pin definitions, observe data**: immutable things are *definitions* (prompt, tool version, schema); everything else is runtime captured in traces.
- **Agent → Generation → Run**: durable identity → immutable pinned lockfile (rollback = flip pointer) → transient episode emitting a trace. Generations are **per-module** — minted on every approve, rolled back via `zz module <m> rollback <n>` (a pointer-flip + content restore). (A whole-Project *checkpoint* composing all modules was built but never wired to any surface, so it was cut 2026-06-22 — roll the Project by rolling each module.)
- **Proposal**: the bridge from observability to a new generation — **always human-approved in v1**.
- **Design B**: live-capture hooks signal + trigger re-capture through the proven parse path; they never build spans.
